Summary

The Middleware & Services course is about the deployment of IoT technologie to obtain information from sensor and then send these informations to actuator, to achieve a completely automated IoT system.

  • The first module is about Cloud Computing and the deployment of virtual machines within the IoT network.
  • The second module concerns the Middleware for IoT in which I discovered the main standards of IoT, such as MQTT and how to create a complete OM2M architecture. I also learnt about the interoperability between diffirent type of sensor , diffirent type of technologies and applications in the IoT world.
  • The third module, is about the deployment of Service Oriented Architecture, we learnt how to develop Web Services by using RESTful API.

Middleware and Services

Cloud Computing and Autonomic Management

This module allowed me to assimilate the concepts of cloud as well as the benefit of implementing an automatic management for the deployed services. After studying the main characteristics that differentiate virtual machines and containers, I was able to rediscover these concepts in practice work, by handling virtual machines with VirtualBox and Docker containers. Then, i focused on automating Docker container by using OpenStack - a cloud service.

I also discovered the SaaS part of cloud computing by implementing an application with microservices architecture on cloud. In the other hand, i also discovered the IaaS part of cloud computing by implementing at the infrastructure level (networking, virtualization) a cloud-based service (with Docker and OpenStack). Middleware for Internet of Things

This module took place in four stages: MQTT protocol, Sierra Wireless MangOH, OneM2M and Node-red. First stage in MQTT i learnt the concept of publisher/subscriber with a broker then i used the MQTT to implement a light detection application with a light sensor, all of that was controled by an Arduino board. In the second stage, i got a chance to take in hand the mangOH cards of Sierra Wireless and their Octave Platform. I was able to use this microcontrollers to detect the presented quantity of CO2 in our classroom.

Third stage, i taught myself about the OneM2M standard and the RESTful architecture, through a MOOC. Then, i carried out practical works to apply the things into an architecture of 3 sensors. I was able to create REST queries to create a resource tree for subscription managing. Finally, i used nodejs and Node-Red to create a gateway between the oneM2M standard and MQTT, with the creation of data entities to light lamps following different MQTT requests arriving on several topics. Service Oriented Architecture

At first, this module took place gradually with the theoretical courses which allowed me to understand the value of service and resource oriented architectures. A comparative approach between REST and SOAP architectures in particular allowed me to know their characteristics and which to choose according to the environment and the expected result. Subsequently, the progressive level tutorials lead to the use of the SpringBoot framework to automate and simplify the depoyment of RESTful API , which then allowed me to create my own web services. More detail about this module can be found in my Automatic room management project.

In this project, my teammate and me created an IoT plartform to automate the management of office room. We asked ourselves which scenarios to be implemented, such as: automatic closing windows, turning on heating, turning off lights .... We wanted to use oneM2M to simulate different sensors/actuators present in the office room. We used RESTful services with SpringBoot to retrieve and process the various data emitted by its simulated sensors to be able to perform actions based on different usecases.
