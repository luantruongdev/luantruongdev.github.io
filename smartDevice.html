Summary

This project is about the fabrication from A to Z of a gas sensor and its deployment in an IoT context. To achieve the main goal of this project, each of these steps need to be realized:

  • AIME laboratory for sensor creation
  • PCB circuit design for sensor integration
  • Software development for the sensor's particular use case.

Smart Devices

AIME internship

AIME (L'Atelier Interuniversitaire de Micro-nano Électronique) is a technological platform accessible to students, universities and industrialists. It's equipped with new technological equipement needed for conception, production and test of silicon integrated circuits. During my 3 days intership at AIME, i was able to get familiar to the whole process of sensor fabrication.

The objective was to realize a gaz sensor based on WO3 nanoparticles. I therefore followed the following different steps:

  • Synthesis of WO3 tungsten trioxide nanoparticles.
  • Development of the micro-electronic chip.
  • Integration of the sensitive layer of nanoparticles on the chip.
  • Electrical characterization of the sensor under controlled atmosphere.
Below is some demonstrated images during this process.

I took part in the realization of these different stages. I was able to see how the nanoparticles used today in components of smart sensor. Several steps in the chip creation process had also been realized by my teammate and me, such as: polysilicon deposition, doping and etching, dry oxide deposition and photo-etching, and metallization. Once the sensors were made, we observed the results using a Scanning Electron Microscope (SEM), in order to verify the quality of our Integration of the sensitive layer of nanoparticles on the chip. Finally, we had our sensors pass a series of electrical tests in order to determine their main characteristics. The final task of this AIME intership is to formulate the gaz sensor's datasheet.

Integration of the gaz sensor into a smart IoT system

In parallel with the development of the connected gas sensor, we also needed to create a Arduino Shiled Circuit under KiCad. For this part, i used some hardware components like an Arduino Uno, an RN2483 chip from Microchip for LoRa connectivity. In the next step, i used LTSpice and KiCad to design all the specifications of the Arduino Uno board.

The PCB design and circuit routing on KiCad was not new to me. KiCad is a powerful tool with a lot of libraries to design a functional circut. KiCad also provided a friendly user interface that facilitate the search of components. With Kicad, i could easily import a basic model of Arduino Uno, then i chose the pin that i want to connect with my shield circuit, i also took in account the floating state of unused pin. Unfortunately, the LTC1050 operational amplifier and the RN2483 was not included in KiCad default libraries. I needed to download the corresponding symbol and then an importation to KiCad library had solved the problem. I also added a potentiometer to imitate the the gas sensor. Finally, i able to create my circuit after adding other components like: resistors and capacitors for each low-pass filter.

To integrate the gas sensor to the main cicuit, i also realized a signal conditionning circuit which is also a impedance matching circuit. This step allowed me to amplify the current and get a much bigger output voltage, which then became the input to my Arduino board.

Below are some images of this process.

The next part was carried out by me and my teammate Michael. First, we connected the gas sensor to the Arduino Uno board in order to retrieve the values ​​that we then sent to the LoRa network, that meant we had to extract the payload from the incoming data frame. After data extraction, we also had to decrypt it and translate it to humain readable data, like value of voltage. Then, thanks to The Things Network and Node-RED, we created a dashboard allowing the user to view the value of the gas sensor and to calibrate it if necessary.
